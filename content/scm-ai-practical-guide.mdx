## AI가 SCM을 바꿀 거라는데

요즘 어디서든 AI 이야기가 나온다. "AI로 수요 예측 정확도가 95%로 올라간다", "공급망 최적화를 AI가 자동으로 해준다", "SCM 업무의 40%가 자동화된다".

정말 그런가?

직접 써봤다. 결론부터 말하면: **마법은 아니다. 하지만 분명히 도움이 되는 영역이 있다.**

중요한 건 AI가 잘하는 것과 못하는 것을 구분하는 것이다. 그래야 시간을 아끼고, 헛된 기대에 실망하지 않는다.

---

## AI가 잘하는 것 3가지

### 1. 데이터 정리와 요약

SCM 담당자의 업무 시간 중 상당 부분이 **데이터를 정리하는 데** 쓰인다.

- ERP에서 뽑은 엑셀 데이터 전처리
- 여러 시트에 흩어진 데이터를 하나로 합치기
- 월간 보고서용 데이터 집계

ChatGPT에게 "이 엑셀 데이터를 피벗 테이블로 정리하는 파이썬 코드 만들어줘"라고 하면 몇 초 만에 나온다. 이전에는 구글링으로 30분 걸리던 일이 2분이면 된다.

**실제 사용 예:**
- "이 CSV에서 품목별 월간 출고량 합계를 구하는 코드"
- "거래처별 납품 리드타임 평균과 표준편차를 계산하는 코드"
- "재고 데이터에서 DOS 6개월 이상인 품목만 필터링"

결과물을 그대로 쓰면 안 되고, 반드시 검증해야 한다. 하지만 **초안을 잡아주는 것**만으로도 시간을 크게 절약한다.

---

### 2. 패턴 분석 보조

데이터를 넣고 "패턴이 있어?"라고 물으면, 사람이 놓치는 것을 잡아줄 때가 있다.

3년간 주간 출고 데이터를 넣었더니, "매년 설 연휴 2주 전에 출고량이 30% 급등하는 패턴이 있습니다"라고 답했다. 알고 있는 패턴이었지만, **정확한 타이밍(2주 전)은 체감과 달랐다.** 체감상으로는 1주 전이라고 생각했는데, 데이터는 2주 전부터 올라간다고 했다.

이런 식으로 AI가 기존 지식을 **보정하거나 정교하게 만들어주는 역할**은 확실히 한다.

**주의할 점:** AI가 말하는 패턴이 통계적으로 유의미한지는 별도 검증이 필요하다. "패턴이 있습니다"라고 말해도, 데이터 포인트가 3개뿐이면 패턴이라고 할 수 없다.

---

### 3. 문서 초안 작성

보고서, 업무 매뉴얼, 거래처 제안서. SCM 담당자가 쓰는 문서는 많다.

"안전재고 재설정 제안 보고서 초안을 써줘. 현황은 이렇고, 문제는 이렇고, 제안은 이렇다"라고 입력하면 **구조 잡힌 초안**이 나온다.

이 초안을 기반으로 수정하는 게 빈 화면에서 시작하는 것보다 훨씬 빠르다.

특히 유용한 경우:
- 영문 메일 작성 (해외 공급업체 커뮤니케이션)
- 데이터 기반 보고서의 서론/결론 작성
- 프로세스 매뉴얼 초안

---

## AI가 못하는 것 3가지

### 1. 맥락 기반 판단

"이 공급업체 납기를 믿어도 될까?"

이 질문에 AI는 답할 수 없다. 해당 업체의 최근 재무 상태, 생산 라인 상황, 담당자와의 관계, 업계 소문 — 이런 **맥락은 시스템 데이터에 없다.**

AI에게 "A업체 vs B업체 중 어디에 발주할까?"라고 물으면, 범용적인 평가 기준을 나열한다. 하지만 "A업체 사장님이 최근에 공장을 증축했는데, 자금 사정이 좋지 않다는 소문이 있다"는 정보는 AI가 알 수 없다.

**공급망의 핵심 의사결정은 여전히 사람의 영역이다.**

---

### 2. 실시간 데이터 접근

ChatGPT는 우리 회사의 ERP에 접속하지 못한다. 오늘의 재고량, 이번 주 출고 계획, 거래처 납품 현황 — 이런 실시간 데이터는 수동으로 제공해야 한다.

매번 CSV를 뽑아서 붙여넣는 것도 한계가 있다. 이걸 해결하려면 API 연동이 필요한데, 대부분의 중소 가구업체에서는 아직 먼 이야기다.

---

### 3. 회사 고유의 비즈니스 룰

"C 거래처에는 화요일에만 발주할 수 있다."
"D 자재는 한 번에 50박스 이상 주문해야 할인이 적용된다."
"배송 기사 김 기사님은 강남 지역만 담당한다."

이런 회사 고유의 규칙은 어디에도 문서화되어 있지 않다. 담당자의 머릿속에 있다. AI는 이 규칙을 모른다.

물론, 이런 규칙을 **명문화해서 AI에게 가르치면** 일부 반영할 수 있다. 하지만 현실적으로 모든 암묵적 규칙을 문서화하는 건 거의 불가능하다.

## 실제로 써본 도구들

### ChatGPT (가장 많이 사용)

- **용도:** 데이터 정리 코드 생성, 보고서 초안, 개념 설명
- **장점:** 범용적이고, 한국어 이해도 좋음
- **한계:** 컨텍스트 길이 제한, 할루시네이션(잘못된 정보 생성)

### 엑셀 Copilot

- **용도:** 엑셀 내에서 직접 수식 생성, 데이터 분석
- **장점:** 데이터를 직접 보면서 질문할 수 있음
- **한계:** 복잡한 분석은 아직 부족

### Python + AI (중급 이상)

- **용도:** 대량 데이터 분석, 시각화, 예측 모델 실험
- **장점:** 자유도가 높고, 반복 작업 자동화 가능
- **한계:** 코딩 지식 필요 (하지만 ChatGPT가 코드를 짜주니까 진입장벽이 낮아짐)

## SCM 업무별 AI 활용도

현시점에서 AI가 실무에 도움이 되는 정도를 솔직하게 평가했다.

| 업무 | AI 활용도 | 현재 수준 | 비고 |
|------|----------|----------|------|
| 발주서 작성 | ★★★☆☆ | 템플릿 기반 자동 생성 가능 | 규칙 기반이라 AI보다 매크로가 나을 수 있음 |
| 수요 예측 | ★★☆☆☆ | 패턴 보조만 가능 | 정확한 예측은 전문 솔루션 필요 |
| 재고 분석 | ★★★★☆ | DOS, ABC 분석 코드 생성 우수 | 데이터만 주면 빠르게 분석 |
| 물류비 분석 | ★★★☆☆ | 비용 배분 로직 설계 도움 | 실 데이터 연동은 수동 |
| 공급업체 평가 | ★★☆☆☆ | 평가 프레임워크 생성 가능 | 정성적 판단은 사람 몫 |
| 보고서 작성 | ★★★★★ | 초안 작성 매우 빠름 | 가장 체감 효과가 큰 영역 |
| S&OP 회의 준비 | ★★★☆☆ | 데이터 집계 + 이슈 정리 | 의사결정은 사람이 해야 함 |
| 프로세스 문서화 | ★★★★☆ | 매뉴얼 초안 생성 우수 | 수정은 필요하지만 속도가 빠름 |

**보고서 작성과 재고 분석**이 현재 가장 실용적인 영역이다. 수요 예측이나 공급업체 평가는 아직 보조적 수준.

## 실무 팁 4가지

### Tip 1: 작은 것부터 시작하라

"AI로 수요 예측을 혁신하겠다"보다, "이번 주 재고 보고서 데이터 정리를 AI로 해보겠다"가 훨씬 현실적이다. 작은 성공 경험이 쌓여야 더 큰 활용으로 나아간다.

### Tip 2: 결과를 반드시 검증하라

AI가 생성한 숫자, 코드, 분석을 그대로 의사결정에 쓰면 안 된다. 반드시 **샘플 체크**를 하라. 10건 중 1건만 직접 계산해서 맞는지 확인하면 된다.

### Tip 3: 프롬프트에 맥락을 넣어라

"재고 분석해줘"보다 "가구업체의 완제품 재고 데이터야. 월 출고량 대비 현재고가 6개월치를 넘는 품목을 찾아줘"가 훨씬 좋은 결과를 낸다. **당신이 알고 있는 맥락을 AI에게 전달해야 한다.**

### Tip 4: 반복 작업을 자동화 대상으로 삼아라

매주 같은 포맷의 보고서를 만든다면, 그 과정을 AI와 함께 자동화하라. 한 번 코드를 만들어 놓으면 다음 주부터는 5분이면 끝난다.

## AI는 도구다

결국 AI는 도구다. 좋은 도구이긴 하다. 하지만 도구가 일을 대신하지는 않는다.

SCM 실무의 핵심은 여전히 **판단력**이다. 이 공급업체를 믿을 수 있는지, 이 예측이 현실적인지, 이 재고를 더 갖고 있어야 하는지. 이런 판단은 현장 경험과 맥락 이해에서 나온다.

AI는 그 판단을 위한 **데이터를 더 빠르게 준비해주는 역할**을 한다. 그리고 그것만으로도 충분히 가치가 있다.
