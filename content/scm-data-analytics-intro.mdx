## 데이터는 쌓이는데 분석은 안 한다

가구업체 SCM 팀장이 한 말이다.

"ERP를 10년째 쓰고 있어요. 데이터는 산더미인데, 의사결정할 때는 여전히 감으로 합니다. 데이터를 보고 싶은데 어디서부터 봐야 할지 모르겠어요."

이 상황이 대부분의 중견 제조업체에서 반복된다. ERP에 발주 이력, 입고 이력, 생산 실적, 재고 현황이 모두 들어 있다. 하지만 그 데이터를 꺼내서 의미 있는 분석으로 연결하는 사람이 없다.

SCM 데이터 분석은 데이터 과학이나 AI를 도입하는 것이 아니다. **이미 있는 데이터에서 패턴을 찾고, 그 패턴으로 더 나은 판단을 하는 것**이다.

---

## SCM에서 분석할 데이터는 4가지 영역

공급망에서 생성되는 데이터를 크게 네 영역으로 나눌 수 있다.

### 1. 수요 데이터

고객 주문, 판매 실적, 수요 예측 관련 데이터.

- 일별/주별/월별 판매 실적
- 품목별, 거래처별, 지역별 판매량
- 주문 접수 시점과 패턴
- 계절성, 트렌드

**핵심 질문**: 앞으로 뭐가 얼마나 팔릴 것인가?

### 2. 공급 데이터

조달, 공급업체, 입고 관련 데이터.

- 공급업체별 납기 준수율
- 발주-입고 리드타임
- 품목별 단가 변동 이력
- 불량률, 클레임 건수

**핵심 질문**: 공급이 안정적인가? 어디서 문제가 생기고 있는가?

### 3. 재고 데이터

재고 수준, 회전율, 과부족 관련 데이터.

- 품목별 현재 재고 수량
- 일별 재고 변동 추이
- 재고 회전율, 재고 일수
- 과잉재고 / 부족재고 품목 리스트

**핵심 질문**: 재고가 적정한가? 돈이 어디에 묶여 있는가?

### 4. 운영 데이터

생산, 물류, 비용 관련 데이터.

- 생산 실적 (계획 대비 실적)
- 라인별 가동률, OEE
- 물류비 (건당, 품목당)
- 총 조달 비용, 재고 유지 비용

**핵심 질문**: 운영이 효율적인가? 비용이 어디서 새는가?

## 분석의 4단계: 무엇이 일어났나 → 왜 → 앞으로 → 어떻게

데이터 분석에는 깊이가 있다.

### 1단계: 기술 분석 (Descriptive) — 무엇이 일어났나

과거 데이터를 정리해서 현황을 파악한다.

"지난 분기 재고 회전율은 4.2회였다."
"A 공급업체 납기 준수율은 78%였다."
"B 품목의 월 평균 판매량은 120개였다."

이것만 해도 상당히 유용하다. 대부분의 회사가 이 단계조차 체계적으로 하지 않고 있다.

### 2단계: 진단 분석 (Diagnostic) — 왜 그런가

숫자 뒤의 원인을 파악한다.

"재고 회전율이 떨어진 이유는 C 품목군의 과잉재고 때문이다."
"납기 준수율이 낮은 이유는 원자재 리드타임이 2주에서 4주로 늘었기 때문이다."

원인을 찾으려면 데이터를 쪼개봐야 한다. 전체 평균이 아니라 품목별, 공급업체별, 기간별로 분해해서 보면 원인이 드러난다.

### 3단계: 예측 분석 (Predictive) — 앞으로 어떻게 될까

과거 패턴으로 미래를 추정한다.

"현재 추세면 다음 달 D 품목은 품절될 가능성이 높다."
"계절 패턴상 6월에 수요가 30% 증가할 것으로 예상된다."

엑셀의 이동평균이나 추세선만으로도 기본적인 예측이 가능하다.

> 수요예측 엑셀 실습은 [수요예측, 엑셀로 시작하기](/demand-forecast-excel)에서 다뤘다.

### 4단계: 처방 분석 (Prescriptive) — 어떻게 해야 하나

분석 결과를 바탕으로 최적의 행동을 제안한다.

"안전재고를 200개에서 280개로 올리면 서비스 레벨 95%를 달성할 수 있다."
"A 공급업체 비중을 60%에서 45%로 줄이고 B 업체를 늘리면 납기 리스크가 감소한다."

이 단계는 시뮬레이션이나 최적화 모델이 필요할 수 있지만, 간단한 What-if 분석은 엑셀로도 가능하다.

## ERP에서 바로 뽑을 수 있는 분석 5가지

복잡한 도구 없이 ERP 데이터만으로 시작할 수 있는 분석들이다.

### 1. 재고 ABC 분석

품목을 구매 금액 기준으로 A/B/C 등급으로 분류. 상위 20% 품목이 전체 금액의 80%를 차지하는 패턴.

**필요 데이터**: 품목 코드, 기간별 구매 금액
**결과**: 어떤 품목에 관리 자원을 집중할지 우선순위

> 상세 방법은 [ABC 분석, 진짜 돈 되는 재고 골라내기](/abc-analysis-practical)에서 다뤘다.

### 2. 공급업체 납기 준수율 추이

공급업체별로 납기 준수율을 월별로 추적.

**필요 데이터**: PO별 약속 납기일, 실제 입고일
**결과**: 어떤 공급업체가 악화되고 있는지 추세 파악

### 3. 수요 변동성 분석

품목별 월 판매량의 표준편차와 변동계수(CV)를 계산.

```
변동계수(CV) = 표준편차 / 평균
```

CV가 높은 품목은 예측이 어렵다. 안전재고를 더 잡아야 한다.
CV가 낮은 품목은 예측이 쉽다. 재고를 최적화할 여지가 있다.

### 4. 재고 일수 분포

품목별 재고 일수(현재 재고 ÷ 일평균 사용량)를 계산해서 분포를 본다.

- 재고 일수 > 90일: 과잉재고 후보 (처분 검토)
- 재고 일수 < 7일: 부족재고 위험 (보충 필요)
- 중간: 정상 범위

### 5. 발주-입고 리드타임 분석

품목별, 공급업체별 실제 리드타임의 평균과 편차.

리드타임이 길수록, 편차가 클수록 안전재고가 많이 필요하다. 리드타임을 줄이는 것이 재고 절감의 가장 효과적인 방법이기도 하다.

## 분석 도구: 무엇을 쓸 것인가

| 도구 | 적합한 상황 | 장점 | 한계 |
|------|------------|------|------|
| **엑셀** | 소규모 분석, 빠른 확인 | 익숙함, 즉시 사용 가능 | 대용량 처리 어려움, 자동화 한계 |
| **피벗 테이블** | 다차원 집계, 패턴 발견 | 코딩 없이 드릴다운 | 시각화 한계 |
| **Power BI / Tableau** | 대시보드, 정기 보고 | 시각화 우수, 자동 갱신 | 학습 비용, 라이선스 |
| **Python / R** | 고급 분석, 예측 모델 | 유연성 최대 | 코딩 필요 |
| **ERP 내장 분석** | 기본 보고서 | 데이터 연동 완벽 | 커스터마이징 한계 |

**시작은 엑셀이면 충분하다.** ERP에서 데이터를 추출해서 피벗 테이블로 보는 것만으로도 상당한 인사이트가 나온다.

정기적으로 봐야 하는 분석이 생기면 Power BI나 대시보드 도구로 자동화한다.

## 분석이 실패하는 이유

데이터 분석을 시작했다가 흐지부지되는 경우가 많다. 패턴을 보면:

**데이터 품질 문제**: ERP 데이터가 정확하지 않다. 입고 처리가 며칠 후에 되거나, 품목 코드가 중복되거나, 단위가 불일치한다. 분석보다 먼저 데이터 정합성을 확인해야 한다.

**분석만 하고 행동 안 함**: 좋은 분석을 했는데 아무도 그걸로 의사결정하지 않는다. 분석의 목적은 행동 변화다. "이 분석 결과로 무엇을 바꿀 것인가"를 먼저 정하고 분석을 시작해야 한다.

**너무 많이 보려고 함**: 처음부터 모든 것을 분석하려고 하면 시작이 어렵다. 가장 돈이 되는 품목 10개, 가장 문제가 많은 공급업체 5곳부터 시작한다.

## 시작은 질문에서

데이터 분석의 출발점은 도구가 아니라 **질문**이다.

"왜 이번 달 결품이 3건이나 났지?"
"어떤 공급업체가 납기를 자꾸 지연하지?"
"어떤 품목에 재고가 과하게 쌓여있지?"

이 질문에 답하려고 데이터를 보는 것이 분석이다. 질문 없이 데이터를 보면 그냥 숫자 구경이 된다.

이번 주에 하나만 해보자. ERP에서 품목별 재고 일수를 뽑아서, 90일 이상인 품목 리스트를 만드는 것. 거기서부터 시작하면 된다.

---

## 정리: 데이터 분석은 거창하지 않다

SCM 데이터 분석에 데이터 과학자가 필요한 게 아니다.

- ERP에 이미 데이터가 있다
- 엑셀 피벗 테이블이면 시작할 수 있다
- 핵심은 "무엇을 알고 싶은가"라는 질문이다
- 작게 시작해서 효과가 보이면 확대한다

감으로 하던 의사결정에 숫자 하나를 더하는 것. 그것이 데이터 기반 SCM의 시작이다.
