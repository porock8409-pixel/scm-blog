## ERP를 쓰고 있으면 디지털화가 된 건가

가구업체 SCM 팀장이 경영 세미나에서 돌아와서 한 말이다.

"디지털 트윈, IoT, AI 예측... 다 좋은 얘기인데, 우리 회사는 ERP 마스터 데이터도 엉망인데 그걸 어떻게 하나요?"

맞는 말이다. ERP를 도입한 것과 디지털화가 된 것은 다르다. ERP에 데이터를 입력하는 것은 **전산화**다. 그 데이터를 활용해서 의사결정을 바꾸는 것이 **디지털화**다.

SCM 디지털 전환의 핵심은 기술 도입이 아니다. **데이터가 의사결정에 실제로 쓰이는가**다.

---

## SCM 디지털 기술의 현재 지형

기술 용어가 넘쳐나지만, SCM에서 실제로 의미 있는 기술은 몇 가지로 정리된다.

### 1. 클라우드 ERP / SaaS

기존 온프레미스 ERP를 클라우드로 전환하는 흐름.

**뭐가 달라지나**: 어디서든 접속 가능. 업데이트 자동. 초기 투자 대신 월 구독료.

**가구업체 적용**: SAP, Oracle 같은 대형 ERP는 대기업 얘기다. 중견 가구업체라면 더존, 영림원 같은 국내 클라우드 ERP나, 물류·구매 특화 SaaS를 검토하는 것이 현실적이다.

**판단 기준**: 현재 ERP의 유지보수 비용이 연간 라이선스+서버 비용보다 큰지, 원격 근무·다공장 관리가 필요한지.

### 2. IoT (사물인터넷)

센서를 붙여서 실시간 데이터를 수집하는 기술.

**SCM에서의 활용**:
- **창고**: 온습도 센서로 원단·목재 보관 환경 모니터링
- **물류**: GPS 트래커로 운송 중 위치 추적
- **생산**: 설비 가동 상태 실시간 수집 (OEE 자동 계산)
- **재고**: RFID 태그로 입출고 자동 기록

**가구업체 적용**: 원단 창고의 온습도 관리가 가장 투자 대비 효과가 명확하다. 원단 변색이나 곰팡이 발생을 예방할 수 있다. 생산 라인 설비 모니터링은 가동률이 핵심 이슈인 공장에서 효과적이다.

**판단 기준**: 수작업으로 수집하던 데이터 중 빈도가 높고 오류가 잦은 것부터. "하루에 3번 사람이 온도를 확인하고 기록한다"면 센서로 대체할 수 있다.

### 3. RPA (로봇 프로세스 자동화)

반복적인 사무 작업을 소프트웨어 로봇이 대신 처리하는 기술.

**SCM에서의 활용**:
- 발주서 작성: 발주점에 도달하면 자동으로 PO 생성
- 입고 확인: 송장과 PO를 자동 대조
- 보고서 생성: 매주 월요일 재고 현황 보고서 자동 생성
- 이메일 처리: 공급업체 납기 확인 메일 자동 발송

**가구업체 적용**: 구매팀이 매일 수작업으로 하는 발주서 작성, 입고 대조, 납기 독촉 메일이 RPA 대상이다. 엑셀 매크로에서 시작해서 단계적으로 확장할 수 있다.

**판단 기준**: "매일 같은 형식으로 반복하는 작업"이 있으면 RPA 후보다. 한 사람이 하루 2시간 이상 반복 작업을 한다면 자동화 효과가 있다.

### 4. 디지털 트윈

물리적 공급망을 디지털로 복제해서 시뮬레이션하는 기술.

**SCM에서의 활용**:
- 공장 레이아웃을 디지털로 구현해서 공정 변경 시뮬레이션
- 공급망 네트워크를 모델링해서 거점 배치 최적화
- 수요 변동 시나리오별 재고 시뮬레이션

**가구업체 적용**: 솔직히 말하면, 대부분의 중견 가구업체에 디지털 트윈은 아직 이르다. 기본 데이터 정합성이 갖춰지지 않은 상태에서 디지털 트윈을 구축하면 "엉망인 현실을 디지털로 복제"하는 것에 그친다.

**판단 기준**: ERP 데이터 정합성이 95% 이상이고, 시뮬레이션이 필요한 복잡한 의사결정(다공장 배치, 대규모 공급망 재설계)이 있을 때.

### 5. AI/ML 기반 예측

인공지능과 머신러닝을 활용한 수요 예측, 이상 탐지.

**SCM에서의 활용**:
- 수요 예측 정확도 향상 (과거 데이터 + 외부 변수)
- 공급업체 리스크 조기 감지
- 재고 최적화 자동 추천
- 품질 이상 패턴 탐지

**가구업체 적용**: AI 예측이 효과를 내려면 최소 2~3년의 깨끗한 데이터가 필요하다. "데이터가 있다"와 "쓸 수 있는 데이터가 있다"는 다르다. 엑셀 이동평균으로 예측하는 것과 AI 예측의 정확도 차이가 의미 있으려면 데이터 양과 품질이 뒷받침되어야 한다.

> 수요예측의 기초는 [수요예측, 엑셀로 시작하기](/demand-forecast-excel)에서 다뤘다.

**판단 기준**: "엑셀 이동평균으로 해도 결과가 비슷하다"면 AI가 필요 없다. 데이터가 복잡하고 변수가 많아서 사람이 패턴을 못 잡을 때 AI가 가치를 낸다.

## 디지털 전환의 단계: 기어가기부터

대부분의 기술 세미나는 최종 단계만 보여준다. 현실에서는 단계적으로 올라가야 한다.

### Level 0: 전산화 이전

종이 대장, 구두 발주, 엑셀 파일이 여기저기 흩어져 있다. 데이터가 시스템에 없다.

**해야 할 것**: ERP 도입 또는 기초 데이터 전산화.

### Level 1: 전산화 (Digitization)

ERP에 데이터가 입력된다. 발주, 입고, 재고가 시스템에 기록된다. 하지만 보고서는 여전히 사람이 엑셀로 만든다.

**해야 할 것**: 데이터 정합성 확보. 입력 지연 줄이기. 마스터 데이터 정리.

### Level 2: 디지털화 (Digitalization)

데이터가 자동으로 가공되어 의사결정에 쓰인다. 대시보드가 있고, 발주점 도달 시 알림이 오고, 정기 보고서가 자동 생성된다.

**해야 할 것**: 대시보드 구축. 자동 알림 설정. 반복 업무 자동화 (RPA).

> SCM 대시보드 설계는 [SCM 대시보드: 한 화면으로 공급망을 보다](/scm-dashboard-design)에서 다뤘다.

### Level 3: 디지털 전환 (Digital Transformation)

데이터가 프로세스 자체를 바꾼다. 수요 예측 기반 자동 발주, 공급업체 성과 기반 자동 배분, 실시간 가시성으로 선제적 대응.

**해야 할 것**: AI/ML 도입. 시스템 간 자동 연동. 조직 역량 강화.

대부분의 중견 가구업체는 Level 1과 Level 2 사이에 있다. **Level 2를 확실히 하는 것이 현실적인 목표**다.

## 가구업체 디지털 전환 로드맵

### 1단계: 데이터 기반 다지기 (6개월)

기술을 도입하기 전에 데이터를 정리한다.

- ERP 마스터 데이터 정합성 점검 (품목 코드 중복, 단위 불일치 등)
- 입고 처리 지연 해소 (실물 입고와 시스템 입고의 시차를 1일 이내로)
- 핵심 품목 50개의 데이터 품질 확보
- 엑셀 기반 주간 대시보드 시작

이 단계를 건너뛰면 이후 모든 투자가 효과를 못 낸다. "쓰레기가 들어가면 쓰레기가 나온다(Garbage In, Garbage Out)"는 데이터 분석의 기본 원칙이다.

### 2단계: 자동화와 가시성 (6~12개월)

검증된 데이터 위에 자동화를 올린다.

- Power BI 등으로 SCM 대시보드 자동화
- 발주점 알림 자동화 (ERP 설정 또는 RPA)
- 주간 보고서 자동 생성
- 공급업체 납기 준수율 자동 집계

비용이 크지 않고 효과가 바로 보이는 자동화부터 시작한다.

### 3단계: 고도화 (12개월 이후)

기본이 갖춰진 후에 고급 기술을 검토한다.

- 수요 예측 모델 도입 (충분한 데이터 축적 후)
- IoT 센서 도입 (ROI가 명확한 영역부터)
- 공급업체 포털 구축 (납기 확인, 발주 자동화)
- 시스템 간 API 연동 (ERP ↔ WMS ↔ 물류사)

## 디지털 전환에서 흔히 하는 실수

### 1. 기술부터 도입한다

"블록체인으로 공급망 추적을 하겠다"고 시작하면 실패한다. 기술은 문제를 해결하는 도구일 뿐이다. **"어떤 문제를 풀 것인가"가 먼저**다.

"납기 지연이 잦다" → 원인이 뭔가 → 공급업체 리드타임 데이터가 없다 → 리드타임 측정부터 시작 → 데이터가 쌓이면 분석 → 분석 결과로 공급업체 전략 조정

이 순서가 맞다. 기술은 중간에 필요할 때 들어간다.

### 2. 한 번에 전부 바꾸려 한다

"ERP도 바꾸고, WMS도 도입하고, 대시보드도 만들고" — 동시에 하면 아무것도 안 된다. 프로젝트가 너무 커서 일정이 밀리고, 현업이 따라가지 못하고, 결국 돈만 쓴다.

**한 번에 하나씩.** 가장 효과가 큰 것부터. 성공 경험이 쌓여야 다음 단계를 추진할 동력이 생긴다.

### 3. IT 부서에만 맡긴다

SCM 디지털 전환의 주체는 SCM 팀이어야 한다. IT는 기술 구현을 지원하는 역할이다. "무엇을 볼 것인가, 어떤 의사결정을 바꿀 것인가"는 현업이 정해야 한다.

IT 주도로 만든 대시보드는 현업이 안 본다. 현업이 원하는 화면과 IT가 만든 화면이 다르기 때문이다.

### 4. 조직 변화를 무시한다

새 시스템을 도입해도 사람들이 기존 방식(엑셀, 전화, 메모)을 고수하면 소용없다. **기술 도입의 절반은 사람을 바꾸는 것**이다.

"왜 이 시스템을 쓰는 게 더 좋은지"를 설명하고, 사용법을 교육하고, 기존 방식을 차단하는(엑셀 보고서를 더 이상 받지 않는) 과정이 필요하다.

## 투자 대비 효과가 높은 디지털화 3가지

중견 가구업체가 당장 시작할 수 있고 효과가 명확한 디지털화.

### 1. 자동 발주 알림 (비용: 거의 없음)

ERP에서 발주점을 설정하고, 재고가 발주점 이하로 떨어지면 이메일이나 메신저로 알림. 대부분의 ERP에 이 기능이 있는데 안 쓰고 있는 경우가 많다.

> 발주점 계산은 [엑셀로 하는 재고 시뮬레이션](/excel-inventory-simulation)에서 다뤘다.

**효과**: 결품 예방, 긴급 발주 감소.

### 2. 대시보드 자동화 (비용: Power BI 월 $10/사용자)

주간 회의용 현황판을 자동으로 갱신. 매주 월요일 아침에 데이터가 자동으로 업데이트되어 있으면, 30분짜리 현황 파악이 1분으로 줄어든다.

**효과**: 회의 효율화, 이상 조기 감지.

### 3. 발주서 자동 생성 (비용: RPA 또는 엑셀 매크로)

정기 발주 품목의 발주서를 자동 생성. 사람은 확인하고 승인만 하면 된다.

**효과**: 구매 담당자 시간 절약, 발주 누락 방지.

이 세 가지를 하는 데 대규모 투자가 필요 없다. 하지만 SCM 팀의 일하는 방식이 확실히 달라진다.

## 정리: 디지털 전환은 데이터를 쓰는 것

SCM 디지털 전환의 본질은 기술이 아니라 **데이터가 의사결정에 쓰이는 것**이다.

- 데이터가 정확해야 한다 (전산화)
- 데이터가 가공되어 보여야 한다 (대시보드)
- 데이터가 행동을 바꿔야 한다 (자동화, 예측)

복잡한 기술을 도입하기 전에, 지금 ERP에 있는 데이터가 정확한지 먼저 확인하자. 그리고 그 데이터를 주간 회의에서 실제로 보고 의사결정에 반영하자.

거기서부터가 디지털 전환이다.
