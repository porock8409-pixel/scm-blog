## 2. 타겟 독자 페르소나

### 페르소나 A: "SCM 신입 김대리"
- **배경**: 대기업/중견기업 SCM 부서 신입~2년차
- **고민**: 현장 용어를 모르겠고, 업무의 큰 그림이 안 보임
- **원하는 것**: 기초 개념을 현장 맥락과 함께 배우기
- **검색 키워드**: "SCM이란", "S&OP 뜻", "안전재고 계산"

### 페르소나 B: "이직 준비하는 박과장"
- **배경**: 타 직군(영업, 생산) 3~5년차, SCM 직무로 전환 고려
- **고민**: SCM이 뭘 하는 건지, 내 경력에 어떻게 연결되는지
- **원하는 것**: SCM 전체 그림 + 각 영역 실무 소개
- **검색 키워드**: "SCM 직무", "물류관리사", "SCOR 모델"

### 페르소나 C: "컨설팅 뛰는 이컨설턴트"
- **배경**: 경영 컨설팅 주니어, SCM 프로젝트 배정
- **고민**: 빠르게 SCM 도메인 지식을 잡아야 함
- **원하는 것**: 프레임워크, 핵심 KPI, 업계 best practice
- **검색 키워드**: "SCOR 프레임워크", "SCM KPI", "공급망 최적화"

## 5. Post 8 아웃라인 (다음 발행 예정)

### "SCOR 모델: SCM의 공통 언어"

```
1. 도입: 왜 공급망에 '공통 언어'가 필요한가
   - 부서마다 다른 용어, 다른 기준 → 소통 비용
   - SCOR = Supply Chain Operations Reference

2. SCOR 6대 프로세스
   - Plan: 수요/공급 균형 맞추기
   - Source: 조달 계획 실행
   - Make: 생산 계획 실행
   - Deliver: 주문 관리, 배송
   - Return: 반품/역물류
   - Enable: 비즈니스 규칙, 데이터 관리
   (각 프로세스에 현장 예시 1개씩)

3. SCOR의 3단계 구조
   - Level 1: 프로세스 유형 (전략적)
   - Level 2: 구성 카테고리 (전술적)
   - Level 3: 프로세스 요소 (운영적)

4. 실무에서 SCOR을 어떻게 쓰나
   - 벤치마킹 도구로서의 SCOR
   - 컨설팅 프로젝트에서의 활용
   - 한계와 현실적 조언

5. 마무리: 다음 글 예고 (SCM KPI)
```

## 6. 콘텐츠 운영 원칙

1. **SEO-first**: 제목에 핵심 키워드 포함, 소제목에 관련 키워드 배치
2. **실무 연결**: 모든 개념에 "현장에서는..." 섹션 포함
3. **시각적 설명**: 가능하면 도식/표 활용
4. **시리즈 연결**: 글 말미에 이전/다음 글 링크
5. **발행 주기**: 주 1회 목표
6. **길이**: 1,500~2,500자 (읽기 시간 5~8분)
